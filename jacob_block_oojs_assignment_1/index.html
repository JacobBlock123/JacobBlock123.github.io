Object Oriented Coding Concepts

By Jacob Block

Classes:

Classes are templates for creating objects that store certain values regarding behaviour and/or initial values for any created objects. All created objects with a given class will share the properties given by the class.

Objects:

An object is something that contains state and behvaiour and can often correspond to real world things. Classes can store multiple values, behaviours, and even other objects.

Constructors:

A constructor is a special subroutine that which will create a new object when called. It will often take in arguments that it will use as values in the created object.

Access Methods:

Access methods handle the retrieval and storage of data and provide an API to perform these services.

Interface:

An interface is a programming structure that can enforce certain properties on an object/class so that an object would have to have a specific property.

Abstraction:

Abstraction is used to hide any background details about something from a user so that they can only see the necessary data and information.

Encapsulation:

Encapsulation bundles data with methods that operates on the data. This hides the values and states of an object in a class.

Inheritance:

Inheritance is where one object inherits the attributes and methods of another class. The class which the first class inherits its attributes and methods from is called the parent class.

Polymorphism:

Polymorphism allows programs to process objects in different ways based off of their data type or their class and the ability to redefine methods for derived classes.

Design Patterns:

Design patterns are strategies for solving commonly found problems. Design patterns make your code more flexible reusable, and maintainable.
